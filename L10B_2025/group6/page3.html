Robotic DevOps Engineer

Robotic DevOps Engineer — це спеціаліст, який поєднує практики DevOps зі сферою робототехніки. Основна мета — автоматизувати розробку, тестування, розгортання та моніторинг робототехнічного програмного забезпечення, забезпечуючи швидкий цикл релізів і стабільність систем у продакшні.

Основні обов’язки

  • Налаштування процесів CI/CD для репозиторіїв із кодом роботів (ROS-пакети, алгоритми обробки даних, прошивки для мікроконтролерів).
  • Автоматизація збірки й тестування робототехнічних компонентів (симуляція у Gazebo або інших середовищах, юніт-тести для драйверів сенсорів).
  • Розгортання прошивок і контейнеризованих сервісів на фізичні роботизовані платформи чи віртуальні середовища (Docker, Kubernetes, AWS RoboMaker).
  • Впровадження інфраструктури як коду (Infrastructure as Code) — Terraform, Ansible, щоб забезпечити відтворювані середовища для девелопменту та продакшну.
  • Налаштування моніторингу й логування роботів у реальному часі (Prometheus, Grafana, ELK Stack) для швидкого виявлення аномалій і відлагодження поведінки.
  • Оптимізація конвеєра релізів: масштабовані пайплайни у GitLab CI, Jenkins або GitHub Actions із розгортанням на edge-пристроях.

Інструменти та технології

  • Контейнеризація: Docker, Podman
  • Оркестрація: Kubernetes, K3s, AWS EKS
  • CI/CD: Jenkins, GitLab CI/CD, GitHub Actions
  • Інфраструктура як код: Terraform, Ansible, Helm Charts
  • Моніторинг: Prometheus, Grafana, ELK Stack (Elasticsearch, Logstash, Kibana)
  • Система контролю версій: Git
  • Хмарні сервіси: AWS (RoboMaker, EC2, S3), Azure (Azure IoT Edge), Google Cloud (Cloud Build, Cloud Run)
  • Робототехнічні фреймворки: ROS 1/2, Gazebo, MoveIt

Робота Robotic DevOps Engineer охоплює як бекенд-розробку сервісів і конфігураційні скрипти, так і взаємодію з роботизованою апаратурою — від тестування сенсорів до розгортання стабільних прошивок. Завдяки автоматизації можна швидко інтегрувати нові модулі, зменшувати час від розробки до продакшну та підтримувати безперебійне функціонування роботизованих систем у виробничому середовищі.
